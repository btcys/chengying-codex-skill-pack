# MERGE_PLAN.md

## 合线计划

> 仅多线程开发启用。
> 长期线程按主线顺序推进；临时线程完成后必须合并并归档，不长期悬挂。

## 合线顺序

1. Core / Design System
2. 基础页面
3. 业务功能
4. 核心复杂模块 / 领域服务
5. 预览/发布
6. 修复与清理

## 合线条件

- [ ] typecheck 通过
- [ ] lint 通过
- [ ] build 通过
- [ ] 无无关文件修改
- [ ] CHANGELOG 已更新
- [ ] TASKS 已更新
- [ ] 风险说明已输出
- [ ] 可回滚

## 临时线程收口

- [ ] 已确认该线程属于短期任务，不继续承担长期职责
- [ ] 已完成合并，或已明确放弃并记录原因
- [ ] 已在 THREADS.md 标记状态与结果
- [ ] 已记录归档时间、回滚点、相关主线分支
- [ ] 已关闭后续任务入口，避免继续在旧临时线程上叠加需求
