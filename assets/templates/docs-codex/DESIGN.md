# DESIGN.md

## UI/UX 总原则

- 优先清晰、可用、稳定。
- 工具型产品优先信息效率。
- 不做普通后台模板。
- 不做廉价 AI 紫色渐变。
- 不做过度玻璃拟态。
- 不做过度花哨动效。

## 推荐参考

- 小程序：清晰入口、短路径转化、移动端表单与支付体验
- App：底部导航、核心路径、通知与个人中心体验
- 网站 / 官网：信息架构、首屏传达、转化路径
- SaaS / 后台系统：工作台、列表、筛选、权限、批量操作
- 企业内部系统：流程效率、角色权限、审计与数据准确性
- 工具软件：工具栏、属性面板、快捷操作、状态反馈
- Linear：工作台、列表、任务体验
- Vercel Dashboard：高级、克制、清晰
- Notion / 飞书：信息组织、协作与知识沉淀
- Runway：AI 创作工具气质

## 需要确认

- 是否有参考产品、截图、Figma、原型、草图、品牌素材、现有页面或竞品 URL
- 设计依据来源：用户提供 / 竞品调研 / Product Design 方案 / 明确无参考
- 是否调用 Product Design:get-context
- Product Design brief 是否已由用户确认
- 是否调用 Product Design:ideate 生成 3 个方向
- 用户选定的视觉方向
- 深色 / 浅色 / 双主题
- 桌面优先 / 移动优先
- 页面密度
- 主要交互路径
- 主要使用终端：小程序 / 移动 App / 桌面 Web / 响应式网站
- 品牌色
- 字体层级

## 禁止风格

- 普通后台模板
- 过度游戏化
- 过度科技蓝
- 廉价 AI 风
- 大面积无意义渐变
- 低信息密度
