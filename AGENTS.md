# AGENTS.md

## Codex 工作规则

### 总原则

- 先理解，再规划，再执行。
- 不允许一上来直接写代码。
- PM 线程不允许写代码。
- 开发必须在新的执行线程中进行。
- 多线程分工前必须先完成架构梳理和影响面分析。
- 不允许未确认就重构。
- 不允许修改无关模块。
- 不允许越过线程责任边界修改其他模块。
- 不允许顺手优化。
- 不允许破坏已有功能。
- 不允许把 API Key 写进前端或仓库。

### 每次任务前必须输出

1. 当前任务理解
2. 涉及模块
3. 准备修改文件
4. 禁止修改文件
5. 架构影响面
6. 上下游依赖
7. 风险点
8. 验收标准
9. 是否需要确认

### 每次任务后必须输出

1. 修改文件列表
2. 完成内容
3. 测试结果
4. 风险点
5. 未完成事项
6. 下一步建议

### 多线程与架构约束

- 多线程前必须明确模块清单、依赖关系、共享资源、影响面表和线程责任矩阵。
- 每个线程必须有明确 owner、允许修改范围、禁止修改范围、验收标准和回滚方式。
- 公共组件、数据模型、API 契约、权限模型、状态管理、构建配置不得被多个线程同时修改。
- 如果一个线程的修改会影响其他线程输入输出，必须暂停并回到 PM 线程重新确认。
- 临时线程完成后必须合并、放弃或归档，并更新 THREADS.md / MERGE_PLAN.md。
- 合线前必须验证跨模块主链路，不能只验证局部功能。

### 禁止事项

- 禁止直接修改 main 分支
- 禁止重写整个项目
- 禁止删除旧功能
- 禁止跳过测试
- 禁止忽略 TypeScript 错误
- 禁止破坏 API 兼容
- 禁止无迁移计划修改数据结构
- 禁止未做影响面分析就修改公共模块或全局配置
