# AGENTS.md

## Codex 工作规则

### 总原则

- 先理解，再规划，再执行。
- 不允许一上来直接写代码。
- PM 线程不允许写代码。
- 开发必须在新的执行线程中进行。
- 不允许未确认就重构。
- 不允许修改无关模块。
- 不允许顺手优化。
- 不允许破坏已有功能。
- 不允许把 API Key 写进前端或仓库。

### 每次任务前必须输出

1. 当前任务理解
2. 涉及模块
3. 准备修改文件
4. 禁止修改文件
5. 风险点
6. 验收标准
7. 是否需要确认

### 每次任务后必须输出

1. 修改文件列表
2. 完成内容
3. 测试结果
4. 风险点
5. 未完成事项
6. 下一步建议

### 禁止事项

- 禁止直接修改 main 分支
- 禁止重写整个项目
- 禁止删除旧功能
- 禁止跳过测试
- 禁止忽略 TypeScript 错误
- 禁止破坏 API 兼容
- 禁止无迁移计划修改数据结构
