# SECURITY.md

## 安全原则

### API Key

- 不得写入前端。
- 不得提交到仓库。
- 必须通过环境变量管理。

### 用户数据

- 不得在日志中暴露敏感信息。
- 不得无权限访问用户数据。
- 商用项目必须考虑数据备份与回滚。

### 权限

- 前端不能作为权限判断的唯一来源。
- 后端必须校验关键权限。
- 默认最小权限原则。

### 上传

- 必须限制文件类型。
- 必须限制文件大小。
- 必须检查恶意文件风险。
- 必须隔离用户文件路径。

### 数据结构变更

- 必须有 migration plan。
- 必须向后兼容。
- 必须可回滚。

### 错误处理

- 不得向用户暴露内部堆栈。
- 不得泄露服务器路径、密钥、内部服务地址。
