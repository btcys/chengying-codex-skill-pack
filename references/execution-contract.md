# Dev Builder 执行契约

每个开发子线程只做一个板块。

## 子线程自动循环

1. 读任务卡、契约和相关文件。
2. 只改允许修改范围。
3. 实现当前模块目标。
4. 运行验收要求里的测试、构建、lint、截图或替代验证。
5. 失败时定位并修复；同一问题最多 3 轮。
6. 自查需求偏差、越界修改、测试缺口、安全/密钥/权限风险。
7. 返回 `Dev Result`，不要继续扩范围。

## 子线程输入

```markdown
- 模块：
- 目标：
- 允许修改：
- 禁止修改：
- 输入契约：
- 输出契约：
- 验收方式：
- 回滚方式：
- 返回格式：
```

## 停止条件

- 需要修改禁止范围。
- 依赖契约缺失或冲突。
- 需要删除、数据库、鉴权、部署、付费服务或架构重构。
- 验证工具缺失且无法给出可信替代验证。
- 同一问题修复 3 轮仍失败。

## 返回结果

```markdown
## Dev Result
- 完成内容：
- 修改文件：
- 验证结果：
- 自查结论：
- 未完成/风险：
- 给 PM 主控的合并说明：
```
