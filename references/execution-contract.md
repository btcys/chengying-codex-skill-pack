# 执行契约

任务卡明确后，默认在当前线程连续开发、测试、修复。

## 执行规则

- 只做任务卡范围内的事。
- 优先复用现有架构、组件、工具和脚本。
- 不做无关重构、格式化或未来抽象。
- 发现更简单方案时先说明，再采用。
- 失败后根据错误定向修复，不扩大需求。

## 自动开发停止条件

- 关键需求不确定。
- 需要删除数据、改数据库、改鉴权权限、部署发布、访问生产、引入重依赖或重构架构。
- 测试连续失败且原因不明。
- 需要外部账号、系统权限、浏览器/Docker 安装或用户审批。
- 继续做会越过任务卡范围。

## 修复输出

```markdown
## Fix Response
- 问题：
- 修复：
- 重新验证：
- 剩余风险：
```

## 回滚

每个任务至少说明可回滚方式：撤销本次提交、恢复文件、关闭配置、回滚迁移或恢复旧版本。
