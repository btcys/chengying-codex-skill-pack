# PRD、设计与阶段计划

不要默认写完整 PRD。只有新正式项目、长期维护、需求边界复杂或用户要求时才写。

## PRD 最小内容

- 目标用户
- 核心场景
- 本次目标
- 非目标
- 主要流程
- 验收标准
- 风险和待确认

## 设计

有用户可见界面时，至少确认：

- 页面/模块结构
- 关键交互和状态
- 参考截图/Figma/现有页面/竞品来源
- 必须对照点和禁止点

没有明确视觉目标时，给 2 到 3 个方向让用户选；不要无限探索。
