# 技术方案 / ADR

不要默认写 TECH_SPEC。只有技术选择会影响长期维护、数据、安全、部署或多人协作时才写。

## 必须补技术方案的情况

- 架构或技术栈选择。
- API 契约、数据模型、数据库迁移。
- 鉴权、权限、支付、上传、客户数据。
- 密钥、Token、外部服务、自动化任务。
- 部署、发布、环境变量、基础设施。
- 公共组件、共享模块、大范围重构。

## 最小结构

```markdown
# TECH_SPEC

## 背景
## 目标与非目标
## 当前现状
## 方案
## 影响范围
## 数据/API/权限/部署
## 风险与回滚
## 验证方式
## 待确认
```

写结论，不写论文。能在任务卡里说清的小技术点，不要单独开 TECH_SPEC。
