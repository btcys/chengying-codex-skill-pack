# 流程模式

本 Skill 不再使用多档流程。

## 统一模式

- 一次性原型、临时试验：不走本流程，直接开发；后续需要长期维护时再接管。
- 正式项目、长期项目、已有项目接管：进入统一正式项目流程。
- 默认当前线程可以继续开发、测试、修复；不要因为流程身份而停止。
- 只有关键不确定、高风险、跨模块、长任务、用户要求或需要独立审查时，才补文档或拆线程。

## 输出强度

| 情况 | 输出 |
| --- | --- |
| 普通明确任务 | 短任务卡 + 开发 + 验证结果 |
| 不确定较多 | 任务卡 + 1 到 3 个关键问题 |
| 跨模块/多阶段 | 短阶段计划 |
| 架构/API/数据/权限/部署/密钥 | TECH_SPEC 或 ADR |
| 长期接管 | CURRENT_STATE |
| 交给其他线程/人 | Handoff |

不要为了流程完整一次性展开所有模板。
