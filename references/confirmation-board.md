# 项目开发前确认台

确认台只用于用户需要逐项确认时，不是必经流程。

## 适用

- PRD、设计、技术方案、阶段计划或 Handoff 有多个待确认项。
- 用户在对话里逐项回复困难。
- Codex 给出多个建议，需要用户选择、采纳或拒绝。

## 不适用

- 普通任务卡已经清楚。
- 用户明确要求直接开发。
- 只是查看项目进度。

## 规则

- 确认台只收集反馈，不替代正式文档。
- 用户选择后，Codex 要把结果同步回任务卡、PRD、TECH_SPEC、PHASE_PLAN、VALIDATION 或 Handoff。
- 用户拒绝时必须写原因；原因是后续修改依据。
- 页面不可用时，降级为 Markdown 表格继续推进。
