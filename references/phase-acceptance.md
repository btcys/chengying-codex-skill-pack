# 验收与打回

本流程不强制每次创建独立验收线程。默认当前线程完成自查；高风险、跨模块、用户要求或需要客观复核时，再独立验收。

## 验收检查

- 任务卡目标是否完成。
- 是否越界修改。
- 测试/构建/lint/截图/接口检查是否运行。
- 未验证项是否写明原因和风险。
- 文档或 `docs/codex/CURRENT_STATE.md` 是否需要更新。

## 通过

```markdown
## 验收结论
- 状态：passed
- 证据：
- 未验证项：
- 风险：
```

## 打回

```markdown
## Fix Request
- 未通过项：
- 证据：
- 期望结果：
- 禁止扩大范围：
- 必须重新验证：
```

修复后按 `Fix Response` 回应，并重新验证。
