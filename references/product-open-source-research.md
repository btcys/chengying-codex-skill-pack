# 产品与开源参考

调研用于帮助复杂项目减少 PRD 偏差，不是所有任务必经流程。

## 何时调研

- 用户要求参考竞品或开源方案。
- 需求方向不清，需要借鉴成熟产品。
- 准备基于开源项目二开。
- 技术选型或商业化风险较高。
- 业务域不熟、功能链路长、多角色/多端/后台、外部 API/数据源多。

## 输出

- 默认输出 `docs/codex/PRODUCT_RESEARCH.md`。
- 默认时间盒 5-15 分钟；超出后仍不确定就标记 `unknown`，不要擅自扩大调研。
- 竞品/同类产品：3-5 个，记录来源、日期、核心路径、信息结构、关键功能、权限/定价/边界、可借鉴点、不照搬点。
- 开源候选：3-5 个，记录来源、日期、地址、License、技术栈、维护状态、覆盖度、二开成本、商用/安全/退出风险。
- 结论：adopt / reference / avoid / unknown，并说明对 PRD、设计、技术路线和 MVP 的影响。

## 规则

- 没有联网、用户禁止联网或资料不足时，明确写未完成调研和原因。
- 不要因为看到开源方案就默认采用。
- 采用开源底座、换技术栈或引入重依赖必须先确认。
- 竞品只能用于对齐需求和体验，不得照搬文案、视觉资产、代码或专有交互。
